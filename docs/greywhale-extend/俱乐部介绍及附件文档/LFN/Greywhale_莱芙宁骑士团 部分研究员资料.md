主要经历：

2052 年，父亲在苏州市虎丘区买下了 120 平的房子，分 360 期还款（这意味着直到 2082 年才能还完房贷），并为一家三口办了苏州户口。自此，杨辉宇开始投靠父母。

三个月后，他已经能够看懂许多全英文资料，他再跟父母要了 2000 块买 *C Primer Plus*、*8086/8088/80286 Assembly Language*、*Writing FreeDOS Programs in C* 等数十本汇编、C 语言全英文书籍（大多是非官方印刷）。

2058 年 11 月 27 日，杨辉宇加入 GSCU，给自己取了一个代号"镰刀_16bit"，当然，他加入的不是 GT，而是 LFN，而且还是同在苏州虎丘的 LFN2312-008 据点。

12 月 7 日，"镰刀_16bit"从一位 SLA 研究员那里借到了 5000 元人民币，他将它们兑换成 464 GSD，在太仓某个由一位 AZ 研究员运营的摊位购买了一台由法兰克福 GT0005-014 据点定制的笔记本电脑"Ryetop 233"（刻有 Demoscene Energy Ultra 的签名），这台笔记本电脑搭载着一颗单核 14 纳米 233 MHz 64 位 RISC-V 处理器（带有一个 80386 硬件仿真模块），RAM 16 MiB，拥有一颗 64 MiB(67.11 MB) 的闪存，操作系统是 FreeDOS + Windows 3.1，内置 4 枚 26650 锂电池（可拆卸），尺寸为 260 x 195 x 48 mm，拥有一块 12.79 英寸 LCD 全面屏，重量 2.8 kg，拥有一个 Type-C PD 快充接口（60 W）和一个 3.5 英寸软驱，结构类似 Nokia N97。

"镰刀_16bit"认为自己应该做点什么有价值的事情，于是，他开始用这台笔记本电脑开发程序（主要是为 FreeDOS 或 Windows 3.1 开发）。2058 年 12 月 31 日，他开发出了他自己的第一款程序 *Sickle Knight 2059*，是一款 2D RPG Roguelike 游戏，程序大小约为 58,000 字节，代码行数约 5.3k LOC，适用于 Windows 3.1，使用 Open Watcom C 编译，开源许可证为 Apache 2.0（他认为，GPLv2 虽然强制开源（Apache 2.0 则不强制开源），但没有明确的专利授权，并且无法合并 Apache 2.0 代码，想用 GPL，那就用 GPLv3，GPLv3 有明确的专利授权，并且还反 Tivoization（但 Apache 2.0 没有反 Tivoization 条款）），在这款游戏中，玩家需要使用镰刀击杀来自四面八方的变异甲壳虫，并且收集各种用于升级镰刀和盔甲的材料让自己不断强大。

2059 年 2 月，他又开发了第二款程序 *SliverFishChem*，程序大小约为 3,500,000 字节，代码行数约 10.3k LOC（后来，他认为 3,500,000 字节还是太大了，于是在 2059 年 12 月重做，对非核心功能进行了简化，并添加了插件功能，重做后代码行数增加到约 36.5k LOC，内置了一个简易的 C89/C99 编译器（不是用来写 DOS 程序的，而是专门用来给《SliverFishChem》写插件的），但大小减少到约 480,000 字节），适用于 MS-DOS 和 FreeDOS，使用 Turbo C 2.0 编译，开源许可证为 GPLv3，是一款化学 3D 建模软件，可用于绘制分子结构建模。

2059 年 7 月，他开发了第三款程序 *Pixals16*，分为 Windows 3.1 版和 DOS 版两个版本，使用 Open Watcom C 编译，核心代码行数约 12.6k LOC，Windows 3.1 版总行数约为 18.6k LOC，大小约 330,000 字节，DOS 版总行数约为 17.2k LOC，大小约 260,000 字节，核心代码采用 Apache 2.0 许可证（他认为，GPLv3 与 iOS 的 App Store 存在条款冲突，核心代码采用 Apache 2.0 是为了方便 iOS 开发者对核心代码进行修改，并为 iOS 开发框架，最终顺利上架到 App Store，整个 App 的源码仍然能以 Apache 2.0 或者除 GPL 以外的其他可合并 Apache 2.0 代码的许可证开源，也可以选择闭源；当然，其他操作系统，包括但不限于 Windows、Android、Linux、FreeBSD、macOS（除 App Store 外。要在 App Store 上架，不要使用 GPL 许可证）、Haiku、OS/2 Warp、Symbian、AmigaOS，无论是否修改了核心代码，都可以选择 Apache 2.0 或者包括 GPLv3 在内的可合并 Apache 2.0 代码的一切许可证（GPLv1/GPLv2 除外），也可以选择闭源），Windows 3.1 版和 DOS 版程序均采用 GPLv3 许可证。

*Pixals16* 是一款类似于 VOCALOID 的程序，它使用的声库非常特殊，是一个后缀为"PST"的大文件（虽然其实通常只有几 MB 甚至数百 KB），程序强制用小端序写入所有数据（他认为这是为了方便其他开发者移植到 PowerPC、Motorola 68000 等大端序平台，当程序在大端序平台运行的时候，声库内部数据仍然可以按小端序读取和写入，小端序平台制作的声库在大端序平台仍然可以直接使用，反之亦然）。

前四个字节（0x0000-0x0003）为标识符"_PST"。

第五个至第六个字节（0x0004-0x0005，u16）为程序主版本号（例如 01 00 表示 1。注意，这是小端序）。

第七个至第八个字节（0x0006-0x0007，u16）为程序次版本号（例如 00 00 表示 0）。

第九个至第十个字节（0x0008-0x0009，u16）声库的基础参数个数（例如 00 01 表示 256），参数个数越多，人声品质越好。

第十一个字节（0x000a）表示基础参数数值的类型（00 表示 i8，01 表示 i16，02 表示 i32，03 表示 i64，10 表示 u8，11 表示 u16，12 表示 u32，13 表示 u64，20 表示 f32，21 表示 f64）。

第十二个至第十三个字节（0x000b-0x000c，u32）表示每个音素的参数个数（例如 10 00 表示 16）。

第十四个字节（0x000d）表示音素参数数值的类型（00 表示 i8，01 表示 i16，02 表示 i32，03 表示 i64，10 表示 u8，11 表示 u16，12 表示 u32，13 表示 u64，20 表示 f32，21 表示 f64）。

第十五个至第十八个字节（0x000e-0x0011，u32）表示音素数量（例如 00 00 01 00 表示 65,536）。

第十九个至第二十六个字节（0x0012-0x0019，u64）表示文件大小（例如 28 01 2C 00 00 00 00 00 表示 32 + 8 + 256 + 12 x 65536 + 2 x 16 x 65536 = 2,883,880 字节）。

第二十七个至第二十八个字节（0x001a-0x001b，u16）表示音素名称长度（例如 0C 00 表示 12 字节）。

第二十九个至第三十二个字节（0x001c-0x001f，u32）表示声库名称长度（例如 08 00 00 00 表示 8 字节）。

接下来是声库名称（强制 UTF-8。实际长度不足给定长度的，后面用 00 填充）。

然后是基础参数，同样的基础参数对应同一人，基础参数差异越大，差别越明显。

再接下来是所有音素的名称（强制 UTF-8。实际长度不足给定长度的，后面用 00 填充）。

最后是每个音素的参数，每个音素的顺序与名称出现先后顺序一一对应。

一个声库的规格用"基础参数数量-基础参数类型-每个音素的参数数量-音素参数类型-音素个数"表示，例如"256-i16-16-i16-65536"表示 256 个 int16 基础参数，每个音素 16 个 int16 基础参数，这个声库有 65536 个音素。

要创建一个声库，先设置基础参数数量、基础参数类型、每个音素的参数数量、音素参数类型，然后录制若干个音素。初次保存后，程序会将各个音素量化为基础参数和各个音素的参数。此后，每次保存都会增量更新基础参数、新音素和被修改的音素的参数。

在使用声库创作歌曲时，歌曲可导出为 MOD 文件（Module 文件）。

2059 年 11 月，他发布了一个"256-i16-16-i16-65536"中文声库，名为"Sichsezn"（源自德语"Sichel Sechzehn"缩写；中文：西克瑟森；英语/德语：Sichsezn；日语：シクセズン），大小为 2,883,880 字节（若使用 3.5 英寸 1.44 MB（1,474,560 字节）软盘安装，需要 2 张软盘而非 3 张，因为在 Windows 3.1 和 DOS，软盘的文件系统是 FAT12，每张软盘格式化为 FAT12 后真正可用的只有 1,457,664 字节，两张软盘总共 2,949,120 字节，实际可用的只有 2,915,328 字节，以 512 字节为 1 个簇，2,883,880 字节实际占用 2,884,096 字节，2,915,328 减去 2,884,096 是 31,232 字节，也就是说，可以先在第一张软盘放置一个不多于 31,232 字节的安装程序，然后把声库文件拆成两部分，第一部分 1,426,216 字节（实际占用 1,426,432 字节），在第一张软盘，第二部分则是剩余的 1,457,664 字节，在第二张软盘），声优就是他本人。
